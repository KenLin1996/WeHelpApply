個人簡介

我是林恩，畢業於輔仁大學哲學系，保持好奇心和積極學習是我的人生態度。 在軟體開發的過程中，將抽象的程式碼轉化為具體的畫面解決問題並實現畫面帶來的成就感是無與倫比的， 我在學習的過程中透過撰寫筆記來整理思緒， 一方面加速自己的學習效率，同時期許未來的自己能夠以清晰、易懂的方式將複雜的技術概念傳達給他人，透過交流與分享激發更多火花。

為了成為軟體工程師，做過什麼努力？

為了成為軟體工程師，我起初在 Udemy 上學習了 Wilson 的「 2023 網頁開發」課程，來學習 HTML、CSS、JavaScript 和 React 等技術， 透過該課程我製作了人生的第一個網頁，並接續完成了圍棋介紹網站和貪吃蛇遊戲等小型專案，這些專案加強了我對技術的理解與掌握。 近期為更深入地了解 React ，我在 Udemy 和 Youtube 等多種資源來學習 React 中的重要概念，如 useState、useEffect 和 router 等， 並完成了一個備忘錄專案。 目前，我正在獨立開發一個食譜網站，雖然這個網站還有許多需要改進的地方， 但這個開發過程讓我更加熟悉 React 技術，並且從中獲得了許多成就感。

是否有想要加入的公司？為什麼想加入該公司？

雖然我對公司的選擇沒有特別偏好，但我期望能加入具實踐敏捷開發的新創公司。儘管我對敏捷開發的了解有限，但我認識到這種方法能夠加速團隊交付成果、強調持續學習和團隊合作，而新創公司通常處於成長和變革的階段，這將為我提供快速學習和成長的機會，刺激我的潛能。因此，我期待進入敏捷開發的新創公司，使我持續不斷地學習和成長。

請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

為了增加自己的面試經驗，我也在人力銀行投遞履歷以尋求面試的機會， 在一次面試中，面試官對於我的作品不斷表示過於簡單，這讓我感到有些挫折，當我詢問他的建議時他的回應卻讓我措手不及：“你有想要進步喔？”這突如其來的一句話讓我感到尷尬和難過。 在回去的路程中，我不斷反思著面試的過程和自己的不足，試著釐清自己受傷的原因並作出調整。對於面試官對我的專案過於簡單的批評，我開始思考如何提升作品的豐富度，於是我試著在專案中引入 firebase 的登入功能試圖增加更多功能和互動性。 同時我也反思面試官對我的學習態度的質疑，我原以為跨領域轉職、自學等過程已能有所證明，但面試官的質疑讓我感到困惑和挫折也讓我了解每個人的認知不同但要讓對方接受才是重點，我決定停止自責開始要尋找解決問題的方法。我認為除了在未來的面試中展示更完善的作品外，還需要積累更多的面試經驗，並向有經驗者尋求適合的面試技巧以更好地展現我的能力及特質。

關於這份申請網頁，分享一個開發時的技術心得

在每個 article 標籤中，我使用一條黑色橫線來清晰地區分標題和我的回答，這條分隔線並非使用 hr 標籤而是透過在每個 h2 標籤後面應用 after 偽元素來實現。這意味著我只需要在 CSS 文件中統一設定 h2 ，而無需在每個 h2 標籤下方都插入 hr 標籤，這種做法不僅節省了重複工作還方便了程式碼的維護。

簡短兩三句話，描述對我最重要的一項人際交往原則

我覺得人際相處最重要的就是誠實，信任和真實的關係才能在溝通中直率地表達自己的想法和感受，也才有長久穩固的關係。在團隊上成員之間要能坦誠地分享想法、問題、狀況才會有助於及時解決問題從而確保任務的成功交付。

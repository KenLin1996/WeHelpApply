個人簡介

我是林恩 ，畢業於輔仁大學哲學系，過往的就業經驗有保險業務、行政人員、倉管等職務，經過這些職業體驗後，我開始思考究竟我想以什麼為我一生的志業，要有不易取代的一技之長、有趣（或具挑戰性）的工作、非體力活的工作（因為老了時候，我根本沒辦法享受生活）、能認識更優秀和更聰明的同事、有可能具有去國外工作的機會。 故我選擇軟體工程師ˋ，符合我對於未來職業的期待，為了成為工程師我辭去工作、上線上課程、 YT 、 IG 都追蹤相關的創作者，去盡所能讓我的生活都充斥軟體工程師相關的資訊。

為了成為軟體工程師，做過什麼努力？

我在 Udemy 上學習了 Wilson 的『 2023 網頁開發 』的課程，我在裡面學習了 HTML 、 CSS 和 JavaScript 的基礎，也跟著課程完成了一個圍棋介紹網站、貪吃蛇遊戲，獨立寫出一個個人網站，並也對於後端有一點簡單的認識。最近為了學習 React 我也在 Udemy 上的『網頁前端框架 React 超入門』建立對於 React 的基本概念，並去 Youtube 看 布魯斯前端 的『 React 快速入門 』用 React 做出一個備忘錄。但我深知依樣畫葫蘆的學習是有限的並開始獨立開發一個食譜網站，儘管這個網站還有很多的不足但這是我學習的一個方式用實作來強化對於技術的熟練度。

如果參與這個訓練，會怎麼安排學習時間？

儘管我自認對於基本的概念有所了解，但我目前還是打算每日花費近十小時作為學習的時間的心理準備。首先是先確定每日的進度的負荷為何，以第一階段的安排為例，會先完成老師給予的目標為主，如果是已經能自主解決（無需從教材中尋找方法）的任務會優先處理，再來才是觀看教學影片，若當週任務提前完成會將第一階段提供的教材提前觀看完畢。
具體時間規劃：
早上精神與專注度較高的時段處理較為複雜的任務（08-14），其餘的時間（15-19）才是用來觀看學習的教材。若當日任務卡關，我會調整兩個小時用於吸收新的資訊以及整理思緒，在過往學習的經驗當中，input 與 output 是需要不斷的平衡與調整。

是否有想要加入的公司？為什麼想加入該公司？

目前沒有特別想過要加入哪間公司，但以類型來說會希望能夠加入新創的公司，因為所引用的技術較大的機率是新的技術（至少不會是快淘汰的技術），在人員的組成也大概率是較為年輕的同事。我覺得軟體工程師這個職業很怕的就是所會的技術無法引進且不能一展所長，而在新創公司我覺得在技術的應用（或接觸）上以及敏捷的開發，會讓我在之後職涯的路會更順遂。

請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

有一次在（前端）面試的經驗當中，面試官不斷的表示我的作品過於簡單，並且在面試結尾我詢問面試官對於我的建議時被當面反問：『你有想要進步喔』，這起面試讓我產生了難過以及自卑等負面狀態。
處理方式：
釐清使我受傷的是什麼話並思考如何解決，對於我的專案過於簡單，我開始思考如何更合理的增加我的專案豐富度，我嘗試著引進 firebase 並使用它的登入功能。接著對於他人質疑我的學習態度，我認知這件事情在短暫的面試就被否定，我是保持高度疑惑因為跨領域轉職、自學等行為，我覺得都已經展現出我在學習態度上的高度熱忱，所以面對這方面的質疑我想我能做出最好的調整，就是在往後的面試中拿出更完善的作品，用實際行動來證明。

關於這份申請網頁，分享一個開發時的技術心得

在每一個 article 中都有一條黑色的橫線來分隔題目與我的回答，我使用的並非是 hr 標籤，而是在每一個 h2 標籤的後面使用 after 這個偽元素，我只要在 css 的檔案中統一設定 h2 即可不需要每一個 h2 下方都打 hr 這樣避免了重工也方便維護程式碼。

簡短兩三句話，描述對我最重要的一項人際交往原則

我覺得人際相處最重要的就是誠實，誠實包含了對他人以及自己誠實，對自己誠實才有可能對他人誠實，也才有可能讓他人真正的幫助到自己。

其他想要說的事情？

我很期待能夠進入 wehelp 跟各位同學們一起學習、成長，在這嚴格的篩選制度下，如果申請上了是對我的認可同時也代表我要更謹慎、負責的去面對接下來的培訓以及我的同儕們也會是具有相當覺悟的夥伴們。高壓環境並且實作導向的培訓方式（模擬未來職場）以及豐富的人脈資源，是我期待進入的 wehelp 的重要原因。
